Explicacin

  1. Pr_train = modelo.predict(X_train)

    Esta lnea genera las predicciones del modelo sobre los datos de entrenamiento.
    El modelo devuelve probabilidades (o logits, dependiendo del loss) para cada muestra en X_train.

    • Pr_train almacena esas predicciones.
    • Cada valor indica la probabilidad de que la muestra pertenezca a la clase positiva.

  1. Pr_test = modelo.predict(X_test)

    Igual que el anterior, pero esta vez se obtienen las predicciones sobre los datos de prueba (X_test).
    Esto permite evaluar qu tan bien generaliza el modelo a datos nuevos que no ha visto antes.

  1. auc_train = roc_auc_score(Y_train, Pr_train)

    Aqu se calcula el AUC (rea Bajo la Curva ROC) para el conjunto de entrenamiento.

    • roc_auc_score mide qu tan bien el modelo es capaz de distinguir entre clases.
    • Cuanto ms cerca est de 1.0, mejor ser la capacidad del modelo de clasificar correctamente.
    • Se usa Y_train como etiquetas verdaderas, y Pr_train como las probabilidades predichas.

  1. auc_test = roc_auc_score(Y_test, Pr_test)

    Similar a la anterior, pero para el conjunto de prueba.

    • Esto es importante porque el AUC en test te dice si el modelo est generalizando bien o si est sobreajustado (overfitting).

  1. print(f'AUC Train: {auc_train}')
    print(f'AUC Test: {auc_test}')

    Finalmente, se imprimen en pantalla los valores de AUC tanto para entrenamiento como para test.

Qu significa un buen AUC?

  • AUC = 0.5 el modelo no distingue entre clases (como lanzar una moneda).
  • AUC > 0.8 el modelo es bastante bueno.
  • AUC 1.0 el modelo clasifica casi perfectamente.

f1 Train: 0.904658934539021

Explicacin

  1. f1_train = f1_score(Y_train, P_train)

    • Esta lnea calcula el F1-score para los datos de entrenamiento.

    • Y_train contiene las etiquetas verdaderas.

    • P_train contiene las predicciones del modelo, ya convertidas en 0 o 1.

    • El F1-score es una mtrica que combina precisin y recall en un solo valor, usando la frmula:

      [ F1 = 2 \cdot \frac{\text{Precisin} \cdot \text{Recall}}{\text{Precisin} + \text{Recall}} ]

    • Es especialmente til cuando tienes clases desbalanceadas o cuando te interesa encontrar un equilibrio entre falsos positivos y falsos negativos.

  1. f1_test = f1_score(Y_test, P_test)

    • Similar a la anterior, pero ahora calcula el F1-score en los datos de prueba.
    • As evalas cmo se comporta el modelo con ejemplos no vistos.
    • Una diferencia muy grande entre f1_train y f1_test puede indicar overfitting.

  1. print(f'f1 Train: {f1_train}')
    print(f'f1 Test: {f1_test}')

    • Estas lneas imprimen los resultados del F1-score para entrenamiento y prueba.
    • Te ayudan a comparar el rendimiento del modelo y saber si est bien entrenado, sobreajustado o subentrenado.

Nota adicional

  • El F1-score oscila entre 0 y 1:
    • 1.0 significa prediccin perfecta.
    • 0.0 indica que no se acert nada.
  • Si tienes un warning de tipo UndefinedMetricWarning, puedes agregar zero_division=0 para evitarlo:
f1_score(Y_train, P_train, zero_division=0)

In[111]:
# Obtener logits y aplicar funcin sigmoide
logits_test = modelo.predict(X_test)
probs_test = 1 / (1 + np.exp(-logits_test))

# Umbrales a evaluar
thresholds = np.arange(0.0, 1.01, 0.05)

# Guardamos las mtricas
precisions = []
recalls = []
f1_scores = []

# Calcular mtricas para cada umbral
for thresh in thresholds:
    preds = (probs_test >= thresh).astype(int)
    precisions.append(precision_score(Y_test, preds, zero_division=0))
    recalls.append(recall_score(Y_test, preds))
    f1_scores.append(f1_score(Y_test, preds, zero_division=0))

# Crear DataFrame
df_metrics = pd.DataFrame({
    'Umbral': thresholds,
    'Precision': precisions,
    'Recall': recalls,
    'F1 Score': f1_scores
})

# Encontrar el umbral con mayor F1
best_index = np.argmax(df_metrics['F1 Score'])
best_threshold = df_metrics.loc[best_index, 'Umbral']
best_f1 = df_metrics.loc[best_index, 'F1 Score']

print(f'Mejor umbral segn F1 Score: {best_threshold:.2f} (F1 = {best_f1:.3f})')
df_metrics.round(3)
