Creamos datos aleatorios

In[2]:
# Total de datos
N = 200
# 200 valores aleatorios entre -5 y 4
X = np.random.random(N)*9 - 5
# Funcin: w=0.5, b=-1, ruido de 0.5 para 200 valores aleatorios con media 0 y desviacin 1
y = (0.5*X - 1) + np.random.randn(N)*0.5

Explicacin

  1. Se define la cantidad total de datos a generar:

N = 200

Esto significa que se van a crear 200 muestras (puntos de datos). Esta variable se usa en las siguientes lneas para definir cuntos valores aleatorios se generarn.

  1. Se generan 200 valores aleatorios en el intervalo entre -5 y 4:

X = np.random.random(N) * 9 - 5

Aqu est lo que ocurre:

  • np.random.random(N) genera 200 nmeros aleatorios con distribucin uniforme entre 0 y 1.
  • * 9 escala esos valores al rango [0, 9).
  • - 5 desplaza el rango a [-5, 4).

As que X ser un arreglo de 200 valores aleatorios distribuidos uniformemente entre -5 y 4.

  1. Se generan los valores de y a partir de una relacin lineal con ruido:

y = (0.5X - 1) + np.random.randn(N)0.5

Esto representa una funcin lineal con pendiente 0.5 y ordenada al origen -1, a la que se le suma ruido aleatorio para simular datos reales. Desglosemos:

  • 0.5 * X - 1 es una funcin lineal: y = 0.5x - 1.
  • np.random.randn(N) genera 200 valores con distribucin normal (media 0, desviacin estndar 1).
  • 0.5 ajusta el nivel del ruido: lo reduce para que tenga una desviacin estndar de 0.5.

Entonces, y se compone de:

  • Una parte determinstica (la lnea recta).
  • Una parte aleatoria (el ruido), que simula variaciones naturales o errores de medicin.

Explicacin

  1. Se define la capa de entrada del modelo:

i = Input(shape=(1,))

Esto indica que el modelo recibir una sola caracterstica como entrada (por ejemplo, una variable x real).

  • shape=(1,) significa que cada ejemplo de entrada es un nmero de una dimensin (como un valor de x).
  1. Se agrega una capa densa (neurona) que toma la entrada:

x = Dense(1)(i)

  • Se crea una capa densa con una sola neurona (Dense(1)), que recibe la entrada i.
  • Esta capa aplica una operacin lineal:
    ( y = w \cdot x + b ),
    donde w y b son los pesos y sesgo que el modelo aprender.
  1. Se construye el modelo final especificando entrada y salida:

modelo = Model(i, x)

  • Se crea un modelo Keras con i como entrada y x como salida.
  • Esto es til cuando se usa la API funcional de Keras (en lugar de la secuencial).
  1. Se imprime un resumen del modelo:

modelo.summary()

Esto muestra una tabla con:

  • Las capas del modelo.
  • El tipo de cada capa.
  • La cantidad de parmetros entrenables.

Como este modelo tiene solo una neurona, el nmero de parmetros ser 2: el peso (w) y el sesgo (b).

  1. Se compila el modelo, especificando cmo se va a entrenar:

modelo.compile(
loss='mse',
optimizer=Adam(learning_rate=0.1),
metrics=['mae']
)

  • loss='mse': Se usa el error cuadrtico medio (mean squared error), tpico en regresin.
  • optimizer=Adam(learning_rate=0.1): Se usa el optimizador Adam con una tasa de aprendizaje de 0.1.
  • metrics=['mae']: Se aade el error absoluto medio (mean absolute error) como mtrica adicional.

Este modelo est diseado para aprender una relacin lineal simple, como por ejemplo los datos generados con y = 0.5x - 1 + ruido.

Entrenamiento

In[]:
r = modelo.fit(
    X.reshape(-1, 1),
    y,
    epochs=800,
    batch_size=32
)

Explicacin

  1. X.reshape(-1, 1)

Esto reorganiza la forma del array X para que tenga una sola columna.

  • X originalmente tiene forma (200,) (200 valores sueltos).
  • X.reshape(-1, 1) convierte eso en una matriz de 200 filas y 1 columna: forma (200, 1).

Esto es necesario porque el modelo espera entradas bidimensionales, aunque tenga solo una caracterstica.

  1. y

Este es el vector de etiquetas (los valores reales de salida), que usamos para entrenar al modelo. No hace falta cambiar su forma porque Keras acepta vectores 1D como salidas.

  1. epochs=800

Se entrena el modelo durante 800 pocas.

  • Una poca significa pasar todos los datos de entrenamiento una vez por el modelo.
  • Entrenar durante muchas pocas ayuda al modelo a aprender mejor, pero tambin puede provocar sobreajuste si es excesivo.
  1. batch_size=32

Durante cada poca, los datos se dividen en lotes (batches) de 32 ejemplos a la vez.

  • Esto hace que el entrenamiento sea ms eficiente y estable.
  • En cada lote, el modelo actualiza los pesos.
  1. r = ...

La variable r guarda el historial del entrenamiento.

  • r.history contiene los valores de la funcin de prdida (loss) y la mtrica (mae) en cada poca.
  • Puedes usarlo luego para graficar la evolucin del entrenamiento.

modelo.layers[1].get_weights()

[array([[0.50980544]], dtype=float32), array([-1.0232817], dtype=float32)]

Explicacin

Esta lnea se usa para obtener los pesos entrenados del modelo, especficamente de la primera capa oculta (que en este caso es tambin la nica capa densa del modelo).

  1. modelo.layers

Es una lista que contiene todas las capas del modelo.
En este ejemplo, el modelo tiene dos capas:

  • layers[0] la capa de entrada: Input(shape=(1,))
  • layers[1] la capa densa: Dense(1)
  1. modelo.layers[1]

Accede especficamente a la capa densa, que es donde estn los pesos (w) y el sesgo (b) que se ajustan durante el entrenamiento.

  1. .get_weights()

Devuelve una lista con dos elementos:

  • El primer elemento es un array con el peso (w) forma (1, 1) porque es una entrada y una neurona.
  • El segundo elemento es un array con el sesgo (b) forma (1,).
